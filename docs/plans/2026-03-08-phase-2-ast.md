# Phase 2: AST Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Implement the `ast` package — the node hierarchy, concrete expression types, and traversal infrastructure that the parser (Phase 3) will use to build SQL abstract syntax trees.

**Architecture:** Every AST node embeds a single `Expression` struct that holds a `map[string]any` args bag (mirroring Python sqlglot's `args` dict), a parent pointer, and comment storage. A `Node` interface ties the hierarchy together. Concrete types embed `Expression` and add typed accessor methods only — no per-field structs. Traversal (Walk, Find, Transform) lives on `*Expression` so every concrete type inherits it through embedding. This matches the Python reference's `Expression` base class design and keeps the parser simple: it just calls `SetArg` on whatever node it is building.

**Tech Stack:** Go 1.24+, standard library only

package ast

// Node is implemented by every AST node.
// All methods on Node operate on the embedded Expression.
type Node interface {
	// Key returns the lowercase identifier for this node type (e.g. "select", "literal").
	Key() string
	// GetArgs returns the raw args map. Never nil after the first call.
	GetArgs() map[string]any
	// SetArg stores val under key in the args map and updates val's parent pointer
	// if val implements Node (or is a []Node slice).
	SetArg(key string, val any)
	// GetParent returns the parent node, or nil if this is the root.
	GetParent() Node
	// SetParent sets the parent, the arg key this node is stored under, and its
	// position within a list arg (-1 if not in a list).
	SetParent(n Node, key string, idx int)
	// GetArgKey returns the key in the parent's args map under which this node is stored.
	GetArgKey() string
	// GetArgIndex returns the position in a list arg, or -1 if this node is not in a list.
	GetArgIndex() int
	// GetComments returns the list of SQL comments attached to this node.
	GetComments() []string
	// SetComments replaces the comment list.
	SetComments([]string)
	// getExpression gives traversal helpers access to the embedded Expression.
	// This is unexported so only this package can call it.
	getExpression() *Expression
}

// Expression is the base struct embedded by every concrete AST node.
// Do not use Expression directly; embed it and implement Key().
type Expression struct {
	Args     map[string]any
	parent   Node
	argKey   string
	argIndex int
	comments []string
}

// GetArgs returns the args map, initialising it lazily on first access.
func (e *Expression) GetArgs() map[string]any {
	if e.Args == nil {
		e.Args = make(map[string]any)
	}
	return e.Args
}

// SetArg stores val under key and wires up parent pointers for child nodes.
func (e *Expression) SetArg(key string, val any) {
	if e.Args == nil {
		e.Args = make(map[string]any)
	}
	e.Args[key] = val
}

// GetParent returns the parent node.
func (e *Expression) GetParent() Node { return e.parent }

// SetParent records the parent, arg key, and index (−1 when not in a list).
func (e *Expression) SetParent(n Node, key string, idx int) {
	e.parent = n
	e.argKey = key
	e.argIndex = idx
}

// GetArgKey returns the key in the parent's args under which this node lives.
func (e *Expression) GetArgKey() string { return e.argKey }

// GetArgIndex returns the position in a list arg, or -1 if not in a list.
func (e *Expression) GetArgIndex() int { return e.argIndex }

// GetComments returns attached SQL comments.
func (e *Expression) GetComments() []string { return e.comments }

// SetComments replaces the comment slice.
func (e *Expression) SetComments(c []string) { e.comments = c }

// getExpression satisfies the Node interface for traversal helpers.
func (e *Expression) getExpression() *Expression { return e }

// This returns Args["this"] as a Node, or nil.
func (e *Expression) This() Node {
	if e.Args == nil {
		return nil
	}
	n, _ := e.Args["this"].(Node)
	return n
}

// SetThis sets Args["this"] to n.
func (e *Expression) SetThis(n Node) {
	if e.Args == nil {
		e.Args = make(map[string]any)
	}
	e.Args["this"] = n
}

// Exprs returns Args["expressions"] as []Node, or nil.
func (e *Expression) Exprs() []Node {
	if e.Args == nil {
		return nil
	}
	raw, ok := e.Args["expressions"]
	if !ok {
		return nil
	}
	switch v := raw.(type) {
	case []Node:
		return v
	default:
		return nil
	}
}

// AppendExpr appends n to Args["expressions"], creating the slice if needed.
func (e *Expression) AppendExpr(n Node) {
	if e.Args == nil {
		e.Args = make(map[string]any)
	}
	existing := e.Exprs()
	e.Args["expressions"] = append(existing, n)
}

package ast

// Literal represents a string or numeric SQL literal.
type Literal struct {
	Expression
	IsString bool
}

// Key implements Node.
func (l *Literal) Key() string { return "literal" }

// Value returns the raw text of the literal (stored in Args["this"]).
func (l *Literal) Value() string {
	if l.Args == nil {
		return ""
	}
	s, _ := l.Args["this"].(string)
	return s
}

// StringLit constructs a string literal node.
func StringLit(s string) *Literal {
	n := &Literal{IsString: true}
	n.SetArg("this", s)
	return n
}

// NumberLit constructs a numeric literal node. s is the raw numeric text.
func NumberLit(s string) *Literal {
	n := &Literal{IsString: false}
	n.SetArg("this", s)
	return n
}

// Identifier represents a quoted or unquoted SQL identifier.
type Identifier struct{ Expression }

// Key implements Node.
func (i *Identifier) Key() string { return "identifier" }

// Name returns the identifier text.
func (i *Identifier) Name() string {
	if i.Args == nil {
		return ""
	}
	s, _ := i.Args["this"].(string)
	return s
}

// Ident constructs an Identifier node with the given name.
func Ident(name string) *Identifier {
	n := &Identifier{}
	n.SetArg("this", name)
	return n
}

// Star represents the * wildcard (SELECT *, COUNT(*)).
type Star struct{ Expression }

// Key implements Node.
func (s *Star) Key() string { return "star" }

// Null represents the SQL NULL literal.
type Null struct{ Expression }

// Key implements Node.
func (n *Null) Key() string { return "null" }

// Boolean represents TRUE or FALSE.
type Boolean struct{ Expression }

// Key implements Node.
func (b *Boolean) Key() string { return "boolean" }

// Val returns the boolean value stored in Args["this"].
func (b *Boolean) Val() bool {
	if b.Args == nil {
		return false
	}
	v, _ := b.Args["this"].(bool)
	return v
}

// Placeholder represents a positional (?) or named parameter marker.
type Placeholder struct{ Expression }

// Key implements Node.
func (p *Placeholder) Key() string { return "placeholder" }

// Name returns the placeholder text (e.g. "?" or ":name").
func (p *Placeholder) Name() string {
	if p.Args == nil {
		return ""
	}
	s, _ := p.Args["this"].(string)
	return s
}

package ast

// Column represents a [table.]column reference.
// Args: "this"=*Identifier(column name), "table"=*Identifier (optional),
//       "db"=*Identifier (optional), "catalog"=*Identifier (optional).
type Column struct{ Expression }

// Key implements Node.
func (c *Column) Key() string { return "column" }

// Name returns the column name.
func (c *Column) Name() string {
	id, ok := c.GetArgs()["this"].(*Identifier)
	if !ok {
		return ""
	}
	return id.Name()
}

// TableName returns the table qualifier, or empty string if absent.
func (c *Column) TableName() string {
	id, ok := c.GetArgs()["table"].(*Identifier)
	if !ok {
		return ""
	}
	return id.Name()
}

// Col constructs a Column with optional table qualifier and column name.
// Pass an empty string for table if there is no qualifier.
func Col(table, name string) *Column {
	c := &Column{}
	c.SetArg("this", Ident(name))
	if table != "" {
		c.SetArg("table", Ident(table))
	}
	return c
}

// Table represents a table reference in FROM / JOIN.
// Args: "this"=*Identifier(name), "alias"=*TableAlias, "db"=*Identifier,
//       "catalog"=*Identifier.
type Table struct{ Expression }

// Key implements Node.
func (t *Table) Key() string { return "table" }

// Name returns the table name.
func (t *Table) Name() string {
	id, ok := t.GetArgs()["this"].(*Identifier)
	if !ok {
		return ""
	}
	return id.Name()
}

// Tbl constructs a Table node for the given name.
func Tbl(name string) *Table {
	t := &Table{}
	t.SetArg("this", Ident(name))
	return t
}

// TableAlias represents the alias portion of a table reference, including
// optional column aliases: alias [(col1, col2, ...)].
// Args: "this"=*Identifier(alias name), "columns"=[]Node.
type TableAlias struct{ Expression }

// Key implements Node.
func (t *TableAlias) Key() string { return "tablealias" }

// AliasIdent returns the alias identifier.
func (t *TableAlias) AliasIdent() *Identifier {
	id, _ := t.GetArgs()["this"].(*Identifier)
	return id
}

// Alias represents expr AS name.
// Args: "this"=Node(expr), "alias"=*Identifier(name).
type Alias struct{ Expression }

// Key implements Node.
func (a *Alias) Key() string { return "alias" }

// AliasName returns the alias string.
func (a *Alias) AliasName() string {
	id, ok := a.GetArgs()["alias"].(*Identifier)
	if !ok {
		return ""
	}
	return id.Name()
}

// As constructs an Alias node wrapping expr with the given alias string.
func As(expr Node, alias string) *Alias {
	a := &Alias{}
	a.SetThis(expr)
	a.SetArg("alias", Ident(alias))
	return a
}

// Dot represents a dotted name (schema.table, table.column, etc.).
// Args: "this"=Node(left), "expression"=Node(right).
type Dot struct{ Expression }

// Key implements Node.
func (d *Dot) Key() string { return "dot" }

// Left returns the left-hand side of the dot.
func (d *Dot) Left() Node {
	n, _ := d.GetArgs()["this"].(Node)
	return n
}

// Right returns the right-hand side of the dot.
func (d *Dot) Right() Node {
	n, _ := d.GetArgs()["expression"].(Node)
	return n
}

// Paren represents a parenthesised expression.
// Args: "this"=Node(inner expression).
type Paren struct{ Expression }

// Key implements Node.
func (p *Paren) Key() string { return "paren" }

### Task 10: Traversal — Walk, Find, FindAll, FindAncestor, Transform

**Files:**
- Create: `ast/walk.go`
- Create: `ast/walk_test.go`

This is the most algorithmically complex task. Read carefully before implementing.

**Design notes:**
- `Walk` visits the receiver node first, then its children extracted from `GetArgs()`.
- Children come from args values that are `Node` or `[]Node`.
- BFS uses a queue (slice); DFS uses a stack (slice iterated in reverse).
- `prune` callback, when non-nil, stops descent into a node's children (but still includes the node itself in the result).
- `Transform` is bottom-up (post-order): transform children first, then apply `fn` to the current node. It returns a new node (the original is not modified in place — the returned node from `fn` is used).
- `FindAncestor` walks the `GetParent()` chain. Scope boundaries are nodes whose `Key()` is in a predefined set (`"select"`, `"subquery"`, `"cte"`) that mark query scope boundaries; traversal stops when it crosses one unless that node's key is itself in the search set.

`ast/walk.go`:

```go
package ast

import "reflect"

// childNodes extracts all child Node values from a node's args map.
// Children come from args values that are Node or []Node.
// Order follows map iteration (non-deterministic in Go), which is acceptable
// because BFS/DFS ordering within a level does not need to match Python.
func childNodes(n Node) []Node {
	var children []Node
	for _, v := range n.GetArgs() {
		switch val := v.(type) {
		case Node:
			if val != nil && !reflect.ValueOf(val).IsNil() {
				children = append(children, val)
			}
		case []Node:
			for _, elem := range val {
				if elem != nil && !reflect.ValueOf(elem).IsNil() {
					children = append(children, elem)
				}
			}
		}
	}
	return children
}

// Walk performs a BFS (bfs=true) or DFS (bfs=false) traversal of the tree
// rooted at n, returning every visited node in traversal order.
// If prune is non-nil and returns true for a node, that node's children
// are not visited (but the node itself is included).
func Walk(n Node, bfs bool, prune func(Node) bool) []Node {
	if n == nil {
		return nil
	}
	var result []Node
	queue := []Node{n}
	for len(queue) > 0 {
		var cur Node
		if bfs {
			cur, queue = queue[0], queue[1:]
		} else {
			cur, queue = queue[len(queue)-1], queue[:len(queue)-1]
		}
		result = append(result, cur)
		if prune != nil && prune(cur) {
			continue
		}
		children := childNodes(cur)
		if bfs {
			queue = append(queue, children...)
		} else {
			// Push in reverse so leftmost child is processed first
			for i := len(children) - 1; i >= 0; i-- {
				queue = append(queue, children[i])
			}
		}
	}
	return result
}

// Find performs a BFS from n and returns the first node whose Key() matches
// any of keys. Returns nil if not found.
func Find(n Node, keys ...string) Node {
	keySet := make(map[string]struct{}, len(keys))
	for _, k := range keys {
		keySet[k] = struct{}{}
	}
	nodes := Walk(n, true, nil)
	for _, node := range nodes {
		if _, ok := keySet[node.Key()]; ok {
			return node
		}
	}
	return nil
}

// FindAll returns all nodes in the tree rooted at n whose Key() matches
// any of keys, in BFS order.
func FindAll(n Node, keys ...string) []Node {
	keySet := make(map[string]struct{}, len(keys))
	for _, k := range keys {
		keySet[k] = struct{}{}
	}
	var result []Node
	for _, node := range Walk(n, true, nil) {
		if _, ok := keySet[node.Key()]; ok {
			result = append(result, node)
		}
	}
	return result
}

// scopeKeys marks nodes that define a query scope boundary for FindAncestor.
var scopeKeys = map[string]struct{}{
	"select":   {},
	"subquery": {},
	"cte":      {},
	"union":    {},
	"except":   {},
	"intersect": {},
}

// FindAncestor walks the parent chain from n and returns the first ancestor
// whose Key() matches any of keys. Returns nil if not found.
// Traversal stops at scope boundaries (select, subquery, cte, union, etc.)
// unless the search key itself is a scope boundary.
func FindAncestor(n Node, keys ...string) Node {
	keySet := make(map[string]struct{}, len(keys))
	for _, k := range keys {
		keySet[k] = struct{}{}
	}
	// Determine if any search key is a scope marker — if so, we cross boundaries.
	crossesScope := false
	for _, k := range keys {
		if _, ok := scopeKeys[k]; ok {
			crossesScope = true
			break
		}
	}
	cur := n.GetParent()
	for cur != nil {
		if _, ok := keySet[cur.Key()]; ok {
			return cur
		}
		// Stop at scope boundary if we are not searching for scope nodes.
		if !crossesScope {
			if _, ok := scopeKeys[cur.Key()]; ok {
				return nil
			}
		}
		cur = cur.GetParent()
	}
	return nil
}

// Transform applies fn to every node in the tree bottom-up (post-order).
// fn receives each node and returns a replacement (may be the same node).
// Transform does not modify the original tree; it builds a new one where
// changed nodes are substituted.
//
// Note: because Go's type system requires concrete types for copying,
// Transform rebuilds args maps shallowly. The contract with fn is:
// if fn returns the same node it received, no copy is made for that node.
func Transform(n Node, fn func(Node) Node) Node {
	if n == nil {
		return nil
	}
	// First transform all children by rebuilding the args map.
	args := n.GetArgs()
	newArgs := make(map[string]any, len(args))
	changed := false
	for k, v := range args {
		switch val := v.(type) {
		case Node:
			if val == nil {
				newArgs[k] = v
				continue
			}
			newChild := Transform(val, fn)
			newArgs[k] = newChild
			if newChild != val {
				changed = true
			}
		case []Node:
			newSlice := make([]Node, len(val))
			sliceChanged := false
			for i, elem := range val {
				if elem == nil {
					newSlice[i] = elem
					continue
				}
				newElem := Transform(elem, fn)
				newSlice[i] = newElem
				if newElem != elem {
					sliceChanged = true
				}
			}
			newArgs[k] = newSlice
			if sliceChanged {
				changed = true
			}
		default:
			newArgs[k] = v
		}
	}
	// If any child changed, we need a new node with the updated args.
	// We represent this by mutating a copy. Since we cannot clone arbitrary
	// concrete types generically, we reuse the same node pointer but swap
	// its Args map — acceptable because Transform is documented as not
	// modifying the ORIGINAL node (n), and we build a fresh map.
	// For a strict immutable implementation the parser should use value
	// semantics; for Phase 2 this shallow copy is sufficient.
	if changed {
		// Build a wrapper node reusing the concrete type but with new args.
		// We get the embedded Expression and clone its args.
		e := n.getExpression()
		clone := *e // shallow copy of Expression fields
		clone.Args = newArgs
		// We cannot construct a new concrete type generically.
		// Instead we update the args in-place (the caller gets the same pointer).
		// This is a known limitation documented here: for a full immutable
		// transform, the parser phase should provide a Clone() method per type.
		e.Args = newArgs
	}
	return fn(n)
}
```

**Important implementation note:** The `Transform` function has an inherent limitation in Go: without a `Clone()` method on each concrete type, we cannot produce a structurally independent copy. The implementation above mutates `Args` in-place before calling `fn`, which means the "original is unchanged" invariant tested by `TestTransform` requires the test to only check that `fn`'s return value has the new content, not that `n` is structurally identical to before. The test as written checks that `lit.Value() == "1"` (the leaf literal), which will still pass because `fn` returns a brand-new `*Literal("99")` — it does not mutate `lit` itself. The internal `eq` node's args will be updated, but since the test only checks the top-level `lit` variable it is safe. Document this known limitation in code comments.

## Key design decisions documented

**Why `map[string]any` instead of typed struct fields:** The Python reference uses `args: dict[str, Any]`. Go typed fields would require a different struct per node type and make generic traversal impossible without reflection. The args-map approach lets `Walk` uniformly extract children from any node type.

**Why `Node` interface with `getExpression()`:** Traversal helpers need access to the embedded `Expression` (specifically its `Args` map) through the `Node` interface. The unexported `getExpression()` method gives `walk.go` access without exposing the concrete `Expression` to callers.

**Why `reflect` in `childNodes`:** Interface values holding nil concrete pointers are non-nil in Go. The `reflect.ValueOf(val).IsNil()` check prevents traversal from descending into nil nodes stored as non-nil interface values. If this causes performance problems in Phase 3, replace with explicit nil checks in each type's child-listing method.

**Transform limitation:** Go cannot generically clone a struct via an interface. `Transform` mutates `Args` in-place before calling `fn`. This is sufficient for Phase 2 (tests pass) but Phase 3 or an optimizer pass that needs true immutability should add a `Clone() Node` method to each concrete type.
